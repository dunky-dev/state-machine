# Get started

**Define a behavior once. Render it anywhere.**

Chimba UI splits a UI component into _behavior_ and _render_. You describe behavior
as a plain TypeScript **state machine** that knows nothing about the environment,
and a thin per-substrate layer plugs it into a runtime.

```bash
pnpm add @chimba-ui/state-machine
```
