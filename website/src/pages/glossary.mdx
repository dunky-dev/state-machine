# Glossary

Every term and concept in `@chimba-ui/state-machine`, with a one-line meaning and a link to its full doc page.

## Building blocks

| Term           | Meaning                                                                                                                                                                           |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **machine**    | The built service from `machine(config)` ‚Äî exposes `start`/`stop`/`send`/`state`/`context`/`select`. [‚Üí](/api/setup)                                                              |
| **setup**      | The authoring entry point ‚Äî `setup().createMachine(literal)` (lightweight) or `setup<Ctx,Ev>().config(registries).createMachine(config)` (names compile-checked). [‚Üí](/api/setup) |
| **state**      | One of the flat, named situations the machine can be in (exactly one at a time). [‚Üí](/api/states)                                                                                 |
| **transition** | An `on` entry: where an event takes the machine ‚Äî optional `target`, `guard`, `actions`. [‚Üí](/api/states)                                                                         |
| **event**      | The `{ type, ‚Ä¶ }` object you `send()` to drive a transition. [‚Üí](/api/states)                                                                                                     |
| **context**    | The machine's data: one plain object, read directly (`m.context.x`), written via `setContext`. [‚Üí](/api/context)                                                                  |
| **setContext** | The single, batched entry point for writing context (shallow-equal deduped). [‚Üí](/api/context)                                                                                    |
| **send**       | Dispatch an event to the machine; events run to completion. [‚Üí](/api/states)                                                                                                      |

## Transitions & actions

| Term                  | Meaning                                                                                                                               |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------- |
| **guard**             | A predicate that gates a transition ‚Äî return `false` and it doesn't fire. [‚Üí](/api/guards)                                            |
| **and / or / not**    | Guard combinators for composing named guards. [‚Üí](/api/guards)                                                                        |
| **fallthrough**       | An array of transitions for one event; the first whose guard passes wins. [‚Üí](/api/guards)                                            |
| **action**            | A fire-and-forget side-effect run on a transition, in order ‚Äî gets `{context, setContext, event, send, computed}`. [‚Üí](/api/actions)  |
| **act**               | Write-sugar returning a context-writing action ‚Äî `act({ field: value })` instead of the `setContext` wrapper. [‚Üí](/api/actions)       |
| **oneOf**             | Conditional action: variadic `{ guard?, actions }` branches, first passing wins (the action analog of fallthrough). [‚Üí](/api/actions) |
| **entry / exit**      | Actions run when a state is entered / left (any path in or out). [‚Üí](/api/actions)                                                    |
| **run-to-completion** | Events queue: a `send()` inside an action waits until the current transition finishes ‚Äî no re-entrancy. [‚Üí](/api/states)              |

## Time, data & derivation

| Term         | Meaning                                                                                                                         |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------- |
| **after**    | A timed transition ‚Äî fires after a delay while in a state; auto-cancelled on exit. [‚Üí](/api/timers)                             |
| **delay**    | An `after` key: a number of ms, or a named delay from `implementations.delays` (can read context). [‚Üí](/api/timers)             |
| **watch**    | Run actions whenever a context/computed field changes ‚Äî in any state, for the machine's lifetime. [‚Üí](/api/watch)               |
| **computed** | A lazy, memoized value derived from context (or other computeds); recomputes only when a read input changes. [‚Üí](/api/computed) |

## Effects & the platform seam

| Term                | Meaning                                                                                                                                      |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| **effect**          | A side-effect with cleanup, scoped to a state: runs on enter, its returned cleanup runs on exit. [‚Üí](/api/effects)                           |
| **ComponentEffect** | A platform-specific effect declared outside the machine ‚Äî touches the DOM or reads props, calls `send()` into the machine. [‚Üí](/api/effects) |
| **implementations** | The named registry on a config ‚Äî `guards` / `actions` / `effects` / `delays` referenced by string. [‚Üí](/api/guards)                          |

## The view boundary

| Term                             | Meaning                                                                                                                                    |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ |
| **connect**                      | A pure function mapping a machine snapshot ‚Üí the view-facing api (handlers + attributes). [‚Üí](/api/connector)                              |
| **connector**                    | Keeps `connect` live: memoizes the snapshot, makes props a reactive input, wires reactions. [‚Üí](/api/connector)                            |
| **snapshot**                     | The memoized view api the connector serves ‚Äî stable identity until the machine or props change. [‚Üí](/api/connector)                        |
| **setProps**                     | Push new props into the connector (a reactive input; shallow-dedup'd). [‚Üí](/api/connector)                                                 |
| **reaction**                     | A `[selector, callback]` tuple that fires a prop-callback from _outside_ the machine on a value change. [‚Üí](/api/reactions)                |
| **makeReaction**                 | Inference helper for a reaction tuple ‚Äî recovers the selector‚Üícallback `Value` type link. [‚Üí](/api/reactions)                              |
| **bindings**                     | The agnostic event/attr vocabulary `connect` speaks ‚Äî `onPress`, `role`, `describedBy`. [‚Üí](/api/connector)                                |
| **normalize**                    | The per-target step translating bindings ‚Üí real props (`onPress` ‚Üí `onClick`; `aria-*` on web). [‚Üí](/api/connector)                        |
| **the machine never sees props** | The defining rule: a machine is pure behavior; props live only at the edge. [‚Üí](/api/connector)                                            |
| **the edge**                     | Where props/platform meet the machine ‚Äî the connector (reactions) + the target's `ComponentEffect` (platform listeners). [‚Üí](/api/effects) |

## Observing changes

| Term          | Meaning                                                                                                                       |
| ------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **subscribe** | Coarse observation ‚Äî fires on _any_ state/context change (what a `useSyncExternalStore` bridge uses). [‚Üí](/api/subscriptions) |
| **select**    | Fine-grained observation ‚Äî narrows to a slice, fires only when _that value_ changes. [‚Üí](/api/subscriptions)                  |
| **selection** | What `select(...)` returns: a value-deduped view with `.value` + `.subscribe`. [‚Üí](/api/subscriptions)                        |

## Composition & scale

| Term            | Meaning                                                                                                                         |
| --------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| **compose**     | Run several peer machines as one unit (orthogonal regions) ‚Äî the answer to "nested/parallel" without nesting. [‚Üí](/api/compose) |
| **composition** | What `compose(...)` returns: bundled `start`/`stop` plus `sync` + `combine`. [‚Üí](/api/compose)                                  |
| **sync**        | A coarse cross-region rule on a composition ‚Äî runs when any member changes. [‚Üí](/api/compose)                                   |
| **combine**     | A value-deduped selection derived across composition members. [‚Üí](/api/compose)                                                 |
| **tags**        | Labels on states so consumers query a _category_ (`hasTag('visible')`) instead of a name. [‚Üí](/api/states)                      |
| **hasTag**      | Check whether the current state carries a tag. [‚Üí](/api/states)                                                                 |
| **matches**     | Exact-state check ‚Äî `m.matches('open')`. [‚Üí](/api/states)                                                                       |
| **createStore** | A tiny reactive cell (value + listeners) for singleton state _outside_ any one machine. [‚Üí](/api/create-store)                  |
| **store**       | What `createStore(...)` returns: `get` / `set` / `subscribe` (+ optional domain methods). [‚Üí](/api/create-store)                |

## Lifecycle

| Term                  | Meaning                                                                                                                                |
| --------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| **start / stop**      | Boot / tear down the machine ‚Äî effects, watchers, and reactions begin / clean up. [‚Üí](/api/setup)                                      |
| **onStart / onStop**  | Hang start/stop-scoped work off the machine's lifecycle (how the connector wires reactions). [‚Üí](/api/setup)                           |
| **MACHINE_INIT**      | The synthetic event fired when effects/watchers boot on `start()`. [‚Üí](/api/setup)                                                     |
| **context ownership** | One plain object per machine, copied from the config at construction, mutated in place ‚Äî its identity never changes. [‚Üí](/api/context) |
